vLLM & TinyLlama: Memory Architecture Reference

The Autoregressive Inference Pipeline through HBM3e VRAM

In high-performance inference, the bottleneck isn't the raw FLOPs of the GPU; it is the management of the KV Cache. vLLM solves this using PagedAttention, breaking memory into 16-token Blocks to eliminate fragmentation during the Token Processing Lifecycle.

// Memory Footprint Calculation (TinyLlama Example)
Head Dimension (d_h): 64
Precision (FP16): 2 bytes/param
Memory per Head: 64 * 2 = 128 bytes
Memory per Token (8 Heads): 8 * 128 = 1,024 bytes (1 KB)
Total Block Memory (16 Tokens): 16 * 1 KB = 16 KB / Layer

1. The Visual Unit: The KV Block

Each block in vLLM is a fixed-size memory structure holding Key and Value tensors. In a Grouped Query Attention (GQA) model like TinyLlama, each token row contains 8 specialized squares, allocated dynamically during the Autoregressive Inference Pipeline.

The "Square" Analogy
Think of each 128-byte square as a 64-dimensional coordinate in concept space. 4 "Key" squares tell the model where to look (geographic tags), while 4 "Value" squares hold the content (the actual payload).

2. Architectural Workflow: Prefill to Decode

Phase Logic (CUDA Cores) Memory Movement (HBM3e)
Prefill Stage Processes initial input tensor; generates first KV cache activation. Allocates first Block from Free Queue. Writes 128-byte head vectors to VRAM.
Decoding Stage Autoregressive loop; predicts next token via high-bandwidth weight streaming. Fetches previous blocks via Page Table; appends new KV row to current block.
Cache Eviction Monitors VRAM pressure via Block Manager during long-context generation. Swaps KV Blocks from GPU HBM to CPU RAM (4GB Swap Space) to avoid OOM.

3. The vLLM Control Plane & Token Processing Lifecycle

1. The Page Table (CPU Map)

The CPU tracks the Autoregressive Inference Pipeline. It maps logical sequences to Physical Block #500 and Physical Block #12. This maps virtual tensor indices to physical HBM3e addresses.

2. Continuous Batching (Throughput)

Instead of static sync, vLLM allows a request to exit the Token Processing Lifecycle the moment it hits a <STOP> token. The associated Blocks are recycled into the Free Block Queue for concurrent sessions.

3. Layer-Wise Paging

In a 22-layer model, each layer maintains independent block pointers. The vLLM scheduler ensures that during each step of the Autoregressive Inference Pipeline, HBM3e streams the specific blocks for that layer to the Tensor Cores.

4. Summary of Memory Logic

  • Total CUDA Blocks: Based on available VRAM (e.g., ~32,000 blocks for 10.86 GB).
  • Grouped Query Attention: 32 Query heads share 4 Key/Value heads for efficient Autoregressive Inference.
  • Dimension Placement: 64 numbers per head represent conceptual traits learned during pre-training.
  • The H200 Advantage: 141GB of HBM3e increases the block pool for the Token Processing Lifecycle, supporting massive concurrency.
