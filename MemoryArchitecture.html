Autoregressive Inference & KV-Memory Management

1. Tensor Dimensionality & Head Projection

During the Autoregressive Inference Pipeline, each input token is projected into high-dimensional space. For a TinyLlama-style model using Grouped Query Attention (GQA), the dimensions are calculated as discrete vectors stored in HBM3e.

// Dimensions for a single Token activation (per layer)
Query_Heads (Q) = 32
Key_Heads (K) = 4
Value_Heads (V) = 4
Head_Dim (d_h) = 64
Precision = Float16 (2 Bytes)

Total_KV_Memory = (K + V) * d_h * Bytes = (4 + 4) * 64 * 2 = 1,024 Bytes (1.0 KB)

Dimensional Placement (Concept Coordinates)

The 64 numbers in each "Square" (Head) aren't arbitrary; they represent coordinates in a latent space learned during training. These dimensions capture linguistic features (e.g., Dimension 12: Temporal Context, Dimension 44: Animate/Inanimate Binary). During the Token Processing Lifecycle, these dimensions are fetched and compared via the Attention Kernel.

2. Kernel Orchestration: The PagedAttention V2 Logic

vLLM's breakthrough is the abstraction of physical VRAM into logical blocks. This prevents Internal Fragmentation (wasted space within a sequence) and External Fragmentation (holes in VRAM memory map).

4. Warp-Level Optimization & Memory Striding

In the Autoregressive Inference Pipeline, a single GPU Warp (32 threads) processes the attention heads. To achieve maximum throughput on H200's HBM3e, memory access must be Coalesced.

By arranging the 64 dimensions into 128-byte segments, vLLM ensures that a warp can read multiple head vectors in a single memory transaction. This reduces the number of "Instruction Stalls" and keeps the Tensor Cores saturated at peak performance.
